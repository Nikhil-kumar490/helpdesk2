Compiler Designing
1. What is a Compiler, Interpreter, and Assembler?

Compiler: A compiler is a language processing system that converts high-level programming language (HLL) code into low-level machine code at once. It performs a series of steps like lexical analysis, syntax analysis, semantic analysis, code optimization, and code generation. A compiled program can be executed repeatedly without the need for recompilation.

Advantages: It is faster during execution because the compilation is completed beforehand. Errors are detected before runtime.

Example: GCC (GNU Compiler Collection), Clang.

Interpreter: An interpreter processes the source code line-by-line, translating it into machine code and executing it simultaneously. It does not generate an intermediate file, making debugging easier. However, interpreted code is generally slower than compiled code during execution.

Example: Python Interpreter, Node.js.

Assembler: An assembler is a program that converts low-level assembly language code into machine language. It is used to bridge the gap between hardware instructions and human-readable assembly code. Assemblers are essential for system-level programming and hardware programming.

Example: NASM (Netwide Assembler), MASM (Microsoft Macro Assembler).

 2. What is a Language Processing System?

A Language Processing System (LPS) refers to the set of tools and processes that convert high-level programming code into an executable machine code format. It consists of several stages:

  • Preprocessor: Handles directives like #include and macro substitution.
  • Compiler: Translates high-level code into an intermediate or assembly representation.
  • Assembler: Converts assembly code into object code (binary code).
  • Linker: Combines multiple object files and external libraries into a single executable file.

Diagram:

        Source Code --> Preprocessor --> Compiler --> Assembler --> Linker --> Executable File
3. What are the Types of Compilers?
  • Single-pass Compiler: Processes the entire source code in one pass, without backtracking or revisiting code sections. It is fast but limited in optimization capabilities.
  • Multi-pass Compiler: Processes the source code multiple times to optimize and improve the final machine code.
  • Cross Compiler: Generates code for a platform that is different from the platform the compiler is running on. For example, compiling code on a Windows system for an ARM processor.
  • Just-in-Time (JIT) Compiler: Compiles and executes code during runtime, commonly used in virtual machines like JVM (Java Virtual Machine).
  • Incremental Compiler: Recompiles only the modified portions of code to save time during development.
 4. Explain the Structure and Phases of a Compiler.

The structure of a compiler is divided into two main phases: Analysis (Front-End) and Synthesis (Back-End).

Analysis Phase: This phase analyzes the source code and breaks it down into an intermediate representation (IR). It includes:

  • Lexical Analysis: Converts the input source code into a sequence of tokens.
  • Syntax Analysis: Validates the syntactical structure of the code using grammar rules.
  • Semantic Analysis: Checks for semantic correctness, such as type compatibility.

Synthesis Phase: This phase generates optimized machine code from the IR. It includes:

  • Intermediate Code Generation: Produces an intermediate representation like three-address code.
  • Code Optimization: Improves performance by reducing unnecessary operations.
  • Code Generation: Translates IR into machine-level instructions.

Diagram:

        Source Code --> [Front-End] --> Intermediate Representation --> [Back-End] --> Machine Code
5. What are Tokens and Lexemes?

Tokens: Tokens are the smallest syntactic units of a programming language. These include keywords, operators, identifiers, literals, etc.

Lexeme: A lexeme is an instance of a token in the source code. For example, in the statement int x = 10;, "int" is a lexeme of the token type "keyword".

 6. Explain the Role of a Lexical Analyzer.

The lexical analyzer, also known as the scanner, is responsible for reading the source code and breaking it into tokens. It also removes comments, whitespace, and invalid characters.

Diagram:

        Source Code --> [Lexical Analyzer] --> Token Stream
7. What is the Role of a Parser?

A parser checks whether the token stream adheres to the grammatical rules of the programming language. It builds a hierarchical structure called a "parse tree." There are two main types of parsing:

  • Top-Down Parsing: Starts from the root of the tree and works towards the leaves.
  • Bottom-Up Parsing: Starts from the leaves and works towards the root.

Diagram: Parse Tree Example for E → E + T

               E
              / \
             E   T
            / \
           T   +
8. Explain Context-Free Grammars (CFG).

CFGs define the syntax of programming languages in terms of production rules. Each rule specifies how a non-terminal symbol can be replaced by a sequence of terminal and non-terminal symbols.

Example CFG Rule:

        E → E + T
        T → T * F
        F → (E) | id
9. How Do You Remove Ambiguity from Grammar?

Ambiguity occurs when a grammar allows multiple parse trees for the same string. To remove ambiguity, rewrite the grammar to ensure a single parse tree for each string.

Example:

        Ambiguous Grammar:
        E → E + E | id

        Unambiguous Grammar:
        E → E + T | T
        T → id
10. What is Writing Grammars for Programming Languages?

Writing grammars for programming languages involves defining a set of rules that describe the syntactic structure of valid strings in a language. Context-Free Grammars (CFG) are commonly used for this purpose. The process often includes removing left recursion to ensure the grammar is suitable for parsing algorithms.

Removal of Left Recursion: The goal is to transform the grammar into an equivalent form without left recursion, as shown below:

  • Original Grammar: A → Aα | β
  • After Removal:
    • A → βA'
    • A' → αA' | ε
 11. What are Parse Trees and Ambiguity?

A parse tree is a hierarchical structure that represents the syntactic structure of a string according to a given CFG. It is used in syntax analysis to check the grammatical structure of the source code.

Ambiguity: A grammar is considered ambiguous if a single string can produce more than one distinct parse tree. Ambiguity complicates the parsing process and may lead to incorrect interpretation of the code.

Example of Ambiguous Grammar:

        E → E + E | id
12. How to Remove Ambiguity from Grammar?

To remove ambiguity, the grammar is rewritten to ensure that every string in the language has a unique parse tree. This process involves restructuring the rules while preserving the language's expressiveness.

Example:

Ambiguous Grammar:

        E → E + E | E * E | id

Unambiguous Grammar:

        E → E + T | T
        T → T * F | F
        F → id
