Compiler Designing
1. What is a Compiler, Interpreter, and Assembler?

Compiler: A compiler is a language processing system that converts high-level programming language (HLL) code into low-level machine code at once. It performs a series of steps like lexical analysis, syntax analysis, semantic analysis, code optimization, and code generation. A compiled program can be executed repeatedly without the need for recompilation.

Advantages: It is faster during execution because the compilation is completed beforehand. Errors are detected before runtime.

Example: GCC (GNU Compiler Collection), Clang.

Interpreter: An interpreter processes the source code line-by-line, translating it into machine code and executing it simultaneously. It does not generate an intermediate file, making debugging easier. However, interpreted code is generally slower than compiled code during execution.

Example: Python Interpreter, Node.js.

Assembler: An assembler is a program that converts low-level assembly language code into machine language. It is used to bridge the gap between hardware instructions and human-readable assembly code. Assemblers are essential for system-level programming and hardware programming.

Example: NASM (Netwide Assembler), MASM (Microsoft Macro Assembler).

 2. What is a Language Processing System?

A Language Processing System (LPS) refers to the set of tools and processes that convert high-level programming code into an executable machine code format. It consists of several stages:

  • Preprocessor: Handles directives like #include and macro substitution.
  • Compiler: Translates high-level code into an intermediate or assembly representation.
  • Assembler: Converts assembly code into object code (binary code).
  • Linker: Combines multiple object files and external libraries into a single executable file.

Diagram:

        Source Code --> Preprocessor --> Compiler --> Assembler --> Linker --> Executable File
