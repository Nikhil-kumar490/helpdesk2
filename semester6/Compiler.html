Compiler Designing
1. What is the role of a Compiler?
 A compiler translates the entire source code into machine code before execution. It generates an executable file that can be run multiple times without recompilation.
Example: C, C++ (GCC, Clang, MSVC)

2. What is the role of an Interpreter?
 An interpreter translates and executes code line by line, without generating a separate executable file.
Example: Python, JavaScript (Python Interpreter, Node.js)

3. What is the role of an Assembler?
 An assembler converts assembly language code into machine code (binary instructions understood by the CPU).
Example: NASM, MASM

Differences Between Compiler, Interpreter, and Assembler

Feature Compiler Interpreter Assembler
Translation Process Converts the entire source code to machine code at once Converts and executes code line by line Converts assembly language to machine code
Execution Speed Faster (once compiled) Slower (executes line by line) Fast (low-level conversion)
Error Detection Detects all errors at once before execution Stops at the first error Syntax errors in assembly are detected
Example Languages C, C++ Python, JavaScript Assembly Language
